# Changelog

## Unreleased — Tactical room architecture

### Added

- Shared deterministic room-layout derivation consumed by rendering and navigation.
- Corridor-derived wall openings, segmented walls, and doorway frames.
- Room-role floor modules and deterministic tactical obstacles.
- Six route-specific procedural architecture profiles and material palettes.
- Independent remote-state obstacle layouts and wall openings.
- Projectile collision against generated cover for both operatives and hostile processes.
- Enemy correction when a generated spawn overlaps tactical architecture.
- Multi-route architecture determinism and bounds checks in CI.

### Changed

- Generated rooms are no longer floor plates enclosed by uninterrupted wall rectangles.
- Visible obstacles and movement collision now come from the same data contract.
- Route identity now changes temporary architecture and material behavior as well as graph and encounter parameters.
- Procedural route palettes and architecture are explicitly classified as implementation scaffolds rather than finished cultural environment art.

## 0.4.0 — Route-specific Chave Geral processes

### Added

- Four route-assigned major processes: The Auditor, The Seizure Chief, The Route Runner, and The Warden.
- Distinct procedural silhouettes, health, movement, attacks, spawn conditions, and defeat feedback for each process.
- Active field confiscation and recovered-manifest reduction for The Seizure Chief.
- Overlap and bridge relocation behavior for The Route Runner.
- Return-passage lockdown for The Warden.
- Route-board process warnings before authorization.
- Personalized hostile-intervention contracts compatible with every process.
- Persistent total and per-process defeat statistics.
- Route/process compatibility checks in CI.

### Changed

- Major encounter selection now comes from route data instead of always using The Auditor.
- Unresolved major processes can affect extraction, threshold seizure, debrief, contract completion, and headquarters filing.
- Auditor-specific permanent permissions now apply only when The Auditor is the assigned response.
- Temporary antagonist geometry is explicitly tracked as systems-only placeholder material.

## 0.3.0 — Instituto Travessia headquarters

### Added

- Primary headquarters startup and return loop.
- Six route authorizations with deterministic parameter profiles.
- Persistent profile migration, field rank, scrip, run statistics, and remote-object totals.
- Persistent object archive with appraisal, provenance, condition, route identity, local/remote seeds, storage, sale, retention, and buyback.
- Seven permanent institutional permissions.
- Headquarters route board, archive, profile, and permissions interface.
- Temporary original Instituto Travessia and A Chave Geral vector marks.
- Asset manifest separating temporary systems assets from future final art.
- Headquarters economy simulation check in CI.
- Headquarters implementation and decision documentation.

### Changed

- The game now begins at Instituto Travessia rather than immediately opening a field briefing.
- Route selection generates the mission and then opens its briefing.
- Debrief returns to headquarters instead of directly generating another run.
- Profile persistence moved to version 2 with legacy migration.
- Institute retention is now a deterministic per-object roll capped at fifteen percent of each filing.
- Route payout, remote-object value, healing, cooldown, safe-window, seizure, remote-vault, and Auditor behavior can be changed by permanent permissions.

## 0.2.0 — Independent interlace

### Added

- Independently seeded remote room graph.
- Cross-state overlap detection and temporary bridges.
- Traversable remote rooms and corridors.
- Remote combat, shrines, vault, loot, provenance, and contract objectives.
- Dual-state minimap.
- Threshold seizure pressure and remote-object consequences.
- Multi-seed deterministic generation checks.

## 0.1.0 — Playable field loop

### Added

- Four operative kits: Sócrates, Zélia Amato, Lia, and Kindred.
- Combat, abilities, dodges, hazards, loot, contracts, interlace, extraction, payout, and local progression.
